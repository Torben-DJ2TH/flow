# Protocol Audit: Priorities and Work Packages

Status: `integration/upstream-beta` at `2c8c520`

This document records protocol deviations, robustness gaps, and optional
extensions found during a static code audit. It is a planning document, not a
statement of full protocol conformance.

## Credit model

The prices below are planning estimates:

- 1 credit represents roughly one focused engineering hour.
- A package includes implementation, focused tests, `cargo fmt`,
  `cargo build --release`, and required README/config documentation.
- Hardware interoperability tests, long-running RF tests, external account
  provisioning, and work caused by undocumented vendor behavior are not
  included.
- Credits are effort estimates, not a guarantee of platform token or billing
  consumption.

## P0 - Correctness and crash safety

### WP-01 - Correct TETRA SDS UTF-16BE coding scheme

**Price: 3 credits**

Problem:

`build_sds_text_payload()` currently uses coding scheme `0x02` for UTF-16BE.
ETSI EN 300 392-2 table 29.29 assigns `0x1A` to UCS-2/UTF-16BE. This affects
Unicode SDS generated by DAPNET, MeshCom, GeoAlarm, and dashboard actions.

Scope:

- Correct the shared SDS text encoder.
- Consolidate duplicate coding-scheme knowledge where practical.
- Add Latin-1 and UTF-16BE golden-vector tests.
- Check every caller of `build_sds_text_payload()`.

Acceptance:

- Latin-1 messages remain byte-identical.
- Non-Latin messages use `0x1A` and UTF-16BE without a BOM.
- Existing SDS, DAPNET, MeshCom, and GeoAlarm tests pass.

Primary files:

- `crates/tetra-entities/src/tpg2200.rs`
- `crates/tetra-entities/src/cmce/subentities/home_mode_display.rs`

### WP-02 - Remove protocol-triggerable panics

**Price: 13 credits**

Problem:

Unexpected SAP messages, encrypted MAC fragments, extended channel allocation,
and other unsupported wire variants can reach `panic!()` or `unimplemented!()`.
A malformed or merely unsupported radio message must not terminate the base
station.

Scope:

- Inventory reachable panic paths in UMAC, CMCE, MM, MLE, LLC, and TETRA PDU
  parsing.
- Convert protocol-input failures to typed errors, rejection responses, or
  logged packet drops.
- Keep assertions that represent provably internal invariants.
- Add malformed and unsupported-PDU regression tests.

Acceptance:

- Unsupported encryption and channel-allocation variants cannot crash the BS.
- Unexpected external SAP/control input is rejected and logged.
- A focused fuzz/smoke corpus runs without panic.

Primary files:

- `crates/tetra-entities/src/cmce/cmce_bs.rs`
- `crates/tetra-entities/src/umac/umac_bs.rs`
- `crates/tetra-pdus/src/umac/fields/channel_allocation.rs`
- Other parser files identified by the inventory

### WP-03 - Enforce an honest TETRA capability profile

**Price: 8 credits**

Problem:

SYSINFO can advertise migration, AIE, circuit-mode data, SNDCP, or advanced-link
features even when the selected path is incomplete or unsupported.

Scope:

- Define the capabilities actually implemented by FlowStation.
- Validate capability combinations during config loading.
- Prevent unsupported services from being advertised.
- Log the effective capability profile at startup.
- Document experimental versus supported capabilities.

Acceptance:

- Invalid capability combinations fail with a clear config error.
- Every advertised SYSINFO service has a usable implementation path.
- Default behavior remains compatible with current Motorola terminals.

Primary files:

- `crates/tetra-config/src/bluestation/config.rs`
- `crates/tetra-config/src/bluestation/sec_cell.rs`
- `crates/tetra-entities/src/umac/umac_bs.rs`
- `example_config/config.toml`

### WP-04 - Verify inbound EchoLink identity against the directory

**Price: 5 credits**

Problem:

Inbound EchoLink currently trusts the callsign claimed in an SDES packet. The
source IP is not matched against the downloaded EchoLink directory entry.

Scope:

- Match claimed callsign/node and source IP against the current directory.
- Refresh the directory once on mismatch before rejecting.
- Add an explicit policy for direct-IP targets and unavailable directory data.
- Log accepted and rejected identity decisions without exposing credentials.

Acceptance:

- A spoofed allowed callsign from a different IP is rejected.
- A valid station survives a stale-directory refresh.
- Direct-IP behavior is explicit and configurable.

Primary files:

- `crates/tetra-entities/src/net_echolink/mod.rs`

## P1 - Interoperability and operational stability

### WP-05 - EchoLink connected-session liveness

**Price: 5 credits**

Problem:

Only sessions in `Connecting` state time out. A connected peer that disappears
without sending BYE can remain connected indefinitely.

Scope:

- Add configurable SDES/session liveness timeout.
- Release TETRA floor, call state, codec state, and UI state atomically.
- Emit optional Telegram/UI disconnect reason.
- Apply the configured maximum session duration.

Acceptance:

- A vanished peer is removed after the configured timeout.
- BYE and timeout cleanup produce the same final state.
- `max_session_secs` is effective and documented.

### WP-06 - EchoLink jitter buffer and loss handling

**Price: 13 credits**

Problem:

EchoLink RTP sequence numbers are ignored. There is no reordering, duplicate
suppression, jitter buffer, or packet-loss handling.

Scope:

- Track RTP sequence numbers per dialog.
- Add a small configurable jitter/reorder buffer.
- Suppress duplicates and late packets.
- Add conservative packet-loss concealment or silence insertion.
- Expose packet loss, jitter, and late-packet counters in the UI.

Acceptance:

- Reordered and duplicated test packets produce stable ordered audio.
- Packet loss does not corrupt following GSM frames.
- Added buffering remains appropriate for half-duplex operation.

### WP-07 - Key EchoLink/TETRA floor only on audio

**Price: 3 credits**

Problem:

An inbound SDES connection currently requests the TETRA floor before RTP audio
arrives, which can create empty transmissions.

Scope:

- Keep the EchoLink session connected without allocating RF floor.
- Request the floor on the first valid RTP audio burst.
- Buffer the first 200-500 ms while media setup completes.
- Release the floor after configurable audio inactivity.

Acceptance:

- A connected but silent EchoLink station does not key the TETRA group.
- The first spoken syllable is not discarded.
- Existing simplex/P2MP routing remains intact.

### WP-08 - Implement SIP UDP transaction behavior

**Price: 13 credits**

Problem:

The Asterisk integration implements SIP dialogs but not a complete RFC 3261 UDP
transaction layer. Retransmitted INVITEs can receive `486`, CANCEL does not
finish the original INVITE with `487`, and unknown BYE requests receive `200`
instead of `481`.

Scope:

- Add client/server transaction keys and retransmission timers.
- Cache and replay responses for duplicate requests.
- Implement CANCEL/INVITE `200`/`487` sequencing.
- Return `481` for unknown in-dialog requests.
- Add transaction cleanup and timeout tests.

Acceptance:

- RFC-style SIPp scenarios pass for INVITE, CANCEL, ACK, BYE, and packet loss.
- A duplicated INVITE never creates a second TETRA call.
- Asterisk interoperation remains compatible with the current dialplan.

Primary file:

- `crates/tetra-entities/src/net_asterisk/entity.rs`

### WP-09 - Enforce the MeshCom outbound contract

**Price: 5 credits**

Problem:

FlowStation permits 512-character MeshCom messages while the published external
UDP interface specifies a maximum of 150 UTF-8 characters. Destination syntax
is not validated.

Scope:

- Enforce the 150-character limit before sending.
- Validate broadcast `*`, groups `10..99999`, and APRS-style callsigns.
- Return clear UI/API validation errors.
- Generate a stable outbound `msg_id` where supported by the node firmware.

Acceptance:

- Invalid destinations and oversized messages are rejected before UDP send.
- Unicode length is counted consistently.
- Valid current UI messages remain compatible.

Primary file:

- `crates/tetra-entities/src/net_dashboard/server.rs`

### WP-10 - Harden MeshCom ingress and deduplicate messages

**Price: 8 credits**

Problem:

MeshCom accepts UDP JSON from any source, does not validate coordinate ranges,
and does not deduplicate repeated `msg_id` values before forwarding.

Scope:

- Add source-IP/CIDR allowlisting.
- Validate type-specific required fields and coordinate/battery ranges.
- Add bounded, persistent-or-TTL `msg_id` deduplication.
- Keep unknown packet types in a diagnostic log without forwarding.

Acceptance:

- Duplicate packets trigger forwarding and GeoAlarm at most once.
- Invalid coordinates never reach Maps or GeoAlarm.
- Untrusted UDP sources can be rejected by config.

Primary files:

- `crates/tetra-entities/src/net_meshcom/mod.rs`
- `crates/tetra-config/src/bluestation/sec_meshcom.rs`

### WP-11 - Correct DAPNET sender/callsign semantics

**Price: 3 credits**

Problem:

The RWTH transmitter protocol carries RIC and function but no authoritative
sender callsign. FlowStation heuristically extracts a callsign from message
text and displays it as if supplied by the protocol.

Scope:

- Rename the field/UI label to `detected callsign`, or leave it empty.
- Preserve recipient as RIC/function.
- Mark heuristic values explicitly in API and persisted logs.
- Add parsing fixtures for callsign-like timestamps and identifiers.

Acceptance:

- Recipient and detected sender cannot be confused in UI or exports.
- Existing RIC routing is unchanged.

### WP-12 - Persistent semantic DAPNET deduplication

**Price: 5 credits**

Problem:

The current in-memory ID includes the changing 8-bit transport counter.
Replayed content with a new counter, or after restart, is forwarded again.

Scope:

- Build a semantic key from RIC, function, message type, and normalized text.
- Persist a bounded TTL cache.
- Keep the transport message ID separately for ACK handling and diagnostics.

Acceptance:

- A replay after reconnect or restart is not forwarded twice within the TTL.
- Legitimate repeated messages outside the TTL are delivered.

## P2 - Completeness and maintainability

### WP-13 - Harden the DAPNET TCP client

**Price: 5 credits**

Scope:

- Add DNS/address iteration and explicit connect timeout.
- Limit protocol line length.
- Treat server `-` responses as actionable protocol state.
- Set Health to connected only after successful login.
- Add transcript-based reconnect and malformed-line tests.

### WP-14 - Validate TETRA bit-field ranges

**Price: 3 credits**

Scope:

- Validate `system_code`, `colour_code`, `sharing_mode`,
  `ts_reserved_frames`, and other serialized fixed-width values.
- Reject values that would otherwise be silently truncated.
- Add boundary-value config tests.

### WP-15 - Complete optional TETRA SDS addressing

**Price: 13 credits**

Scope:

- Implement or formally reject short-number addressing.
- Add TSI extension and external-subscriber-number handling.
- Define DM-MS behavior.
- Generate the appropriate SDS reports for unsupported destination forms.

Acceptance:

- Every parsed address variant has a deterministic standards-based result.
- Unsupported variants do not disappear silently.

### WP-16 - Align EchoLink configuration and directory presence

**Price: 8 credits**

Scope:

- Give `auto_connect`, reconnect interval, and maximum session duration their
  documented behavior.
- Publish ONLINE, BUSY, and OFFLINE directory states.
- Make callsign/node filters and direct-IP policy explicit.
- Key dialogs by endpoint and identity rather than IP only.

### WP-17 - Protocol conformance and negative-test suite

**Price: 21 credits**

Scope:

- Add ETSI golden bit vectors for supported TETRA PDUs.
- Add parser property/fuzz tests for malformed PDU input.
- Add EchoLink SDES/BYE/RTP transcript fixtures.
- Add MeshCom JSON schema and routing tests.
- Add DAPNET TCP transcript and REST payload tests.
- Add SIPp-compatible call-flow scenarios.

Acceptance:

- The suite covers success, malformed input, retransmission, timeout, and
  restart behavior.
- CI runs the deterministic subset without RF hardware.

## Optional feature packages

These packages are not required for protocol correctness.

### FP-01 - TETRA capability and conformance dashboard

**Price: 8 credits**

Show advertised versus implemented capabilities, active carriers, protocol
profile, unsupported requests, and recent parser rejections in Health.

### FP-02 - PDU capture, replay, and fuzz corpus tooling

**Price: 21 credits**

Add sanitized TETRA PDU capture, deterministic replay, regression corpus
management, and a developer CLI for reproducing field failures without RF
hardware.

### FP-03 - Advanced EchoLink operations

**Price: 21 credits**

Add live QSO roster, directory ONLINE/BUSY display, `oNDATA` chat/info,
configurable DTMF commands, conference policy, audio statistics, and UI
connect/disconnect controls.

### FP-04 - Full MeshCom telemetry dashboard

**Price: 13 credits**

Parse and graph `temp1`, `temp2`, humidity, QFE, QNH, gas, and CO2 data with
retention, export, thresholds, and forwarding rules.

### FP-05 - MeshCom route-quality analysis

**Price: 13 credits**

Build route history from origin/via paths, RSSI/SNR, duplicates, and age. Show
direct versus relayed reception and automatically mark stale nodes offline.

### FP-06 - DAPNET RIC metadata and decoder profiles

**Price: 8 credits**

Maintain names, categories, personal/group/rubric classification, and
configurable ROT-1 or time-message decoders per RIC.

### FP-07 - SIP transaction diagnostics

**Price: 8 credits**

Expose Call-ID, CSeq, transaction state, retransmissions, final response, RTP
peer, and TETRA mapping in the dashboard with a sanitized text export.
